# ELA_LIB 库命名与格式规范

## 一、文件命名
1. 库文件统一使用 `ela_` 前缀，如 `ela_button.c`、`ela_uart.c`
2. 主函数文件不使用 `ela_` 前缀，保持 `elaco_main.c/h`
3. 头文件使用宏防止重复包含，格式：`_ELA_XXX_H_`（全大写），如 `_ELA_BUTTON_H_`

## 二、函数命名
1. 函数名使用 **PascalCase** 命名，格式为 `Module_ActionDetails`
   - `Module`：模块名，首字母大写，如 `Button`、`Uart`、`MT6816`
   - `ActionDetails`：动作描述，**每个单词首字母大写、不使用下划线分隔**
   - 正确：`Uart_SendString`、`MT6816_ReadAngle`、`Queue_IsFull`
   - 错误：~~`Uart_Sendstring`~~（String 未大写）、~~`Queue_Is_Full`~~（不应使用下划线）
2. 需要在主循环或定时器中周期性调用的函数，末尾加 `_proc` 后缀
   - 命名格式为 **全小写** `module_action_proc`，如 `button_scan_proc`
   - 与普通函数的 PascalCase 区分，便于快速识别周期性调用函数
3. HAL 库回调函数（如 `HAL_UART_RxCpltCallback`）保持原命名，不修改

## 三、变量命名
1. **全局变量**：以 `gov_` 开头，如 `gov_rx1_offset`、`gov_tx1_complete`
2. **局部变量**：全小写，无 `gov_` 前缀，如 `tx_data`、`rx_data`
3. **静态变量**：全小写，无特殊前缀，如 `pressed`、`last_state`
4. **结构体全局变量**：以 `gov_` 开头、`_st` 结尾，如 `gov_mt6816_st`，便于识别结构体类型
5. **状态机全局变量**：以 `gov_` 开头、`_fsm` 结尾，如 `gov_motor_mode_fsm`

## 四、函数文档注释格式
每个对外公开的函数必须在定义处添加以下格式的文档注释：

```
/****
 * @ 原型: <函数签名>
 * @ 输入: <参数说明>
 * @ 输出: <返回值说明>
 * @ 说明: <功能描述>
 * @ 注意: <注意事项>（可选）
 ********/
```

- `@ 原型` — 函数签名，包括返回类型和参数类型
- `@ 输入` — 每个参数名 + 冒号 + 说明；多个参数用空格分隔
- `@ 输出` — 返回值和类型说明；void 的函数写 "void"
- `@ 说明` — 功能描述
- `@ 注意` — 可选，特殊注意事项
- 注释以 `/****` 开头，`********/` 结尾
- 静态（static）函数也需要文档注释

## 五、行内注释
1. 变量声明后使用 `//` 加空格接注释，如：`static bool pressed; // 按键按下标志`
2. 代码逻辑注释使用 `//` 单独成行，位于被注释代码上方
3. 关键操作步骤使用 `/* ... */` 风格的块注释

## 六、代码格式
1. **缩进**：统一使用 4 个空格，不使用 Tab
2. **大括号**：换行风格（Egyptian 风格），左大括号不单独换行
3. **换行符**：使用 Windows CRLF 格式
4. **行尾**：不留多余空格
5. **注释分隔**：
   - 任何 `.c` 文件都可用 `/* -N- 标题内容 */` 划分功能区域，N 从 1 开始递增
   - `elaco_main.c` 可配合 `/* -N- 开始 ... */` 和 `/* -N- 结束 ... */` 成对使用
   - 其他 `.c` 文件只需使用 `/* -N- 标题内容 */` 单行标记，不需要成对
   - 示例：`/* -1- 电机状态变量 */`、`/* -2- 静态函数 */`、`/* -3- 公共函数 */`

## 七、头文件包含顺序
头文件按以下顺序包含：
1. 自身的 `elaco_main.h`（中央包含头）
2. ST 官方库头文件（`main.h`、`spi.h`、`usart.h` 等）
3. 标准库头文件（`stdbool.h`、`string.h`、`stdint.h` 等）

## 八、宏定义命名
1. 功能常量：全大写 + 下划线，如 `DMA_BUF_SIZE`、`LONG_PRESS_MS`
2. 掩码：全大写 + `_MASK` 后缀，如 `MT6816_STATUS_MASK`
3. 状态码：全大写 + 前缀，如 `MT6816_STATUS_NORMAL`

## 九、结构体与变量命名
1. **结构体全局变量**：以 `gov_` 开头、`_st` 结尾，如 `gov_mt6816_st`
2. **结构体类型**：使用 `_T` 后缀，且大写，如 `MT6816_ANGLE_T`

## 十、区域分隔标记
1. 任何 `.c` 文件内如有多个模块或功能分区，使用 `/* -N- 标题内容 */` 格式的注释进行分隔
2. N 从 1 开始递增，如 `/* -1- 电机状态变量 */`、`/* -2- 静态函数 */`
3. 此标记不限于主循环文件，适用于所有源文件
4. `elaco_main.c` 可额外使用 `/* -N- 开始 ... */` 和 `/* -N- 结束 ... */` 成对包裹大段分区
