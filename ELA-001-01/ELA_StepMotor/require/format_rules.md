# ELA_LIB 库命名与格式规范

## 一、文件命名
1. 库文件统一使用 `ela_` 前缀，如 `ela_button.c`、`ela_uart.c`
2. 主函数文件不使用 `ela_` 前缀，保持 `elaco_main.c/h`
3. 头文件使用宏防止重复包含，格式：`_ELA_XXX_H_`（全大写），如 `_ELA_BUTTON_H_`

## 二、函数命名
1. 函数名使用 **PascalCase** 命名，格式为 `Module_ActionDetails`
   - `Module`：模块名，首字母大写，如 `Button`、`Uart`、`MT6816`
   - `ActionDetails`：动作描述，**每个单词首字母大写、不使用下划线分隔**
   - 正确：`Uart_SendString`、`MT6816_ReadAngle`、`Queue_IsFull`
   - 错误：~~`Uart_Sendstring`~~（String 未大写）、~~`Queue_Is_Full`~~（不应使用下划线）
3. HAL 库回调函数（如 `HAL_UART_RxCpltCallback`）保持原命名，不修改

## 三、变量命名
1. **全局变量**：以 `g_` 开头，如 `g_rx1_offset`、`g_tx1_complete`
2. **局部变量**：全小写，无 `g_` 前缀，如 `tx_data`、`rx_data`
3. **静态变量**：全小写，无特殊前缀，如 `pressed`、`last_state`
4. **结构体全局变量**：以 `g_` 开头、`_st` 结尾，如 `g_mt6816_st`，便于识别结构体类型
5. **状态机全局变量**：以 `g_` 开头、`_fsm` 结尾，如 `g_motor_mode_fsm`

## 四、函数文档注释格式
每个对外公开的函数必须在定义处添加以下格式的文档注释：

```
/****
 * @ 输入: <参数说明>（可选，有输入的时候写）
 * @ 输出: <返回值说明>（可选，有输出的时候写）
 * @ 说明: <功能描述>
 * @ 注意: <注意事项>（可选）
 ********/
```

- `@ 输入` — 每个参数名 + 冒号 + 说明；多个参数用空格分隔
- `@ 输出` — 返回值和类型说明；void 的函数写 "void"
- `@ 说明` — 功能描述
- `@ 注意` — 可选，特殊注意事项
- 注释以 `/****` 开头，`********/` 结尾
- 静态（static）函数也需要文档注释

## 五、代码格式
1. **缩进**：统一使用 4 个空格，不使用 Tab
2. **大括号**：换行风格（Egyptian 风格），左大括号不单独换行
3. **换行符**：使用 Windows CRLF 格式
4. **行尾**：不留多余空格

## 六、头文件包含顺序
头文件按以下顺序包含：
1. 自身的 `elaco_main.h`（中央包含头）
2. ST 官方库头文件（`main.h`、`spi.h`、`usart.h` 等）
3. 标准库头文件（`stdbool.h`、`string.h`、`stdint.h` 等）

## 七、宏定义命名
1. 功能常量：全大写 + 下划线，如 `DMA_BUF_SIZE`、`LONG_PRESS_MS`
2. 掩码：全大写 + `_MASK` 后缀，如 `MT6816_STATUS_MASK`
3. 状态码：全大写 + 前缀，如 `MT6816_STATUS_NORMAL`

## 八、结构体与变量命名
1. **结构体全局变量**：以 `g_` 开头、`_st` 结尾，如 `g_mt6816_st`
2. **结构体类型**：使用 `_T` 后缀，且大写，如 `MT6816_ANGLE_T`

## 九、函数分层（hlp / drv / usr）
每个 `.c` 文件内的函数必须按照以下三层分组，各组之间用标准分隔符隔开：

### 分层定义
1. **hlp（Helper / 辅助函数）**：不使用硬件底层，纯计算、数据转换、或其他帮助用户函数构建函数的函数。无 HAL 库调用、无寄存器操作。
2. **drv（Driver / 驱动函数）**：包含对硬件底层的初始化、寄存器配置、HAL 库调用等。是对硬件的最基础封装。
3. **usr（User / 用户函数）**：用户直接调用的函数，可以使用芯片底层（HAL 库），是"随时可以掏出来用"的接口函数。

### 分层顺序
文件内函数按以下顺序排列：**hlp → drv → usr**

### 分隔符格式
```c
/* <module> hlp end */
//----------------------------------------------------------------------------------
/* <module> drv start */

...

/* <module> drv end */
//----------------------------------------------------------------------------------
/* <module> usr start */

...

/* <module> usr end */
```

- `<module>` 替换为模块名（小写），如 `can`、`eeprom`、`timer`
- 分隔线 `//----------------------------------------------------------------------------------` 固定 **84 字符宽**（2 个 `/` + 82 个 `-`）
- `/* <module> xxx start */` 放在第一个该层函数之前
- `/* <module> xxx end */` 放在最后一个该层函数之后
- 某层无函数时，该层可省略（如只有 drv + usr 则跳过 hlp 段）

## 十、行宽限制
1. **每行不超过 84 字符**（含注释）
2. 分隔线 `//----------------------------------------------------------------------------------` 即为 84 字符的标准参考线
3. 超长行应通过换行、缩短注释、提取变量等方式处理
4. 每一个.c和.h文件最后需要空一行，否则keil编译会包Warning
